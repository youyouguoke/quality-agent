## 执行流程

1. 使用 MCP zhimi 工具调用 `get_return_overview` 获取整体概况
2. 调用 `get_accept_reason_analysis` 获取受理原因数据
3. 调用 `get_retest_result_analysis` 获取复测结果数据
4. 调用 `get_defect_cause_analysis` 获取不良原因数据
5. 调用 `get_defect_material_analysis` 获取不良物料数据
6. 调用 `get_responsibility_analysis` 获取责任归属数据
7. 调用 `get_state_analysis` 获取处理状况数据
8. 读取 knowledge/质量基线标准.md 对关键指标进行基线对比

## 领域知识

- 复测完成率 = 已复测+待翻新+已翻新+已完成的数量 / 退货总数
- 返修完成率 = 已翻新+已完成的数量 / 退货总数
- 受理原因分四大类：质量（平台商-质量、三包期内换货、产品质量问题、维修、多维换货、开箱损、平台商-质量换货）、7无（7天无理由、平台商-七无）、付费换新（平台商-付费换新）、其他
- 不良原因和不良物料字段可能包含多个值（逗号分隔），需做多值拆分统计
- 空值应排除，仅统计有复测结果的记录

## 数据加工规则

受理原因分类时，按以下归类：
- **质量类**：平台商-质量、三包期内换货、产品质量问题、维修、多维换货、开箱损、平台商-质量换货
- **7无类**：7天无理由、平台商-七无
- **付费换新类**：平台商-付费换新
- **其他**：不在上述分类中的

## 输出格式要求

严格按以下8个章节输出报告：

### 一、整体概况
用表格展示：产品名称、生产工厂、维修工厂、退货数量、复测完成率、返修完成率

### 二、受理原因分析
数据已按四分类汇总，用表格展示各分类的数量和占比，并展示每个分类下的明细原因

### 三、复测结果分析
展示TOP5复测结果的数量和占比，其余归入"其他"

### 四、不良原因分析
展示TOP5不良原因的数量和占比，其余归入"其他"

### 五、不良物料分析
展示TOP5不良物料的数量和占比，其余归入"其他"

### 六、责任归属分析
展示TOP10责任归属的数量和占比，其余归入"其他"

### 七、处理状况分析
用表格展示各状态的数量和占比

### 八、核心结论与建议
基于以上7个维度的数据，总结3-5条核心发现和改善建议。对比基线标准给出正常/预警/严重判定。
