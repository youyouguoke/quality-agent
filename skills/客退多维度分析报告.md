# 客退多维度分析报告

## 元信息
- **版本**: 1.0
- **创建时间**: 2026-04-09
- **最后更新**: 2026-04-09
- **触发条件**: 用户问客退分析、退货分析、客退报告、退货报告、客退概况

## 描述
对客退机数据进行7个维度的结构化分析，生成完整的客退分析报告。

## 知识
- 复测完成率 = 已复测+待翻新+已翻新+已完成的数量 / 退货总数
- 返修完成率 = 已翻新+已完成的数量 / 退货总数
- 受理原因分为四大类：质量（平台商-质量、三包期内换货、产品质量问题、维修、多维换货、开箱损、平台商-质量换货）、7无（7天无理由、平台商-七无）、付费换新（平台商-付费换新）、其他
- 不良原因和不良物料字段可能包含多个值（逗号分隔），数据已做多值拆分统计
- 空值已在统计时排除，仅统计有复测结果的记录

## 流程
1. 调用 `return_overview` 工具获取7个维度的结构化数据
2. 严格按以下8个章节输出报告

## 输出格式
### 一、整体概况
用表格展示：产品名称、生产工厂、维修工厂、退货数量、复测完成率、返修完成率

### 二、受理原因分析
数据已按四分类（质量、7无、付费换新、其他）汇总好，用表格展示各分类的数量和占比，并展示每个分类下的明细原因。

### 三、复测结果分析
数据仅包含有复测结果的记录（排除未复测的），展示TOP5复测结果的数量和占比，其余归入"其他"。

### 四、不良原因分析
仅统计有复测结果的记录。展示TOP5不良原因的数量和占比，其余归入"其他"。注意数据已做多值拆分统计。空值已排除。

### 五、不良物料分析
仅统计有复测结果的记录。展示TOP5不良物料的数量和占比，其余归入"其他"。注意数据已做多值拆分统计。空值已排除。

### 六、责任归属分析
仅统计有复测结果的记录。展示TOP10责任归属的数量和占比，其余归入"其他"。注意数据已做多值拆分统计。

### 七、处理状况分析
用表格展示各状态的数量和占比。

### 八、核心结论与建议
基于以上7个维度的数据，总结3-5条核心发现和改善建议。
