# 异常主动预警

## 元信息
- **版本**: 1.0
- **创建时间**: 2026-04-09
- **最后更新**: 2026-04-09
- **触发条件**: 用户提到预警、告警、异常、巡检、监控、有什么问题、质量风险

## 描述
查询系统自动巡检产生的质量预警告警，帮助用户快速了解当前质量风险状况并给出处理建议。

## 知识
- 系统每4小时自动巡检一次，检查4个维度的质量异常
- 规则1 return_volume_spike：SKU退货量环比增长超30%预警、超50%严重
- 规则2 defect_concentration：单一不良原因占比超40%预警、超50%严重
- 规则3 supplier_iqc_low：供应商IQC批次合格率低于95%预警、低于90%不合格
- 规则4 retest_backlog：复测完成率低于80%预警、低于60%严重积压
- 告警分三级：critical（严重，需立即处理）、warning（预警，需关注）、info（信息）
- 用户可以确认（acknowledge）告警，表示已知悉或已处理

## 流程
1. 调用 `get_alerts` 工具获取当前告警列表
2. 按严重程度排序展示：先 critical，再 warning
3. 对每条告警给出处理建议（可结合 `root_cause_analysis` 或 `baseline_compare` 深入分析）
4. 如用户询问特定告警的详情，可调用相关分析工具进一步追溯

## 输出格式
### 当前预警概况
展示告警统计：未确认告警总数、严重/预警/信息各多少条。

### 严重告警（需立即处理）
逐条展示 critical 告警，包含告警标题、详情、触发时间、建议措施。

### 预警告警（需关注）
逐条展示 warning 告警，包含告警标题、详情、建议措施。

### 处理建议
对最紧急的告警给出具体处理步骤建议。
